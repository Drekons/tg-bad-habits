# ТГ бот для отслеживания вредных привычек

Задача: разработать ТГ бот который будет вести учёт вредных привычек (например курение, употребление алкоголя и т.п.)

### Технологии

- Golang
- MySQL
- Telegram Bot API

### Инфраструктура

- Бот деплоится на отдельный сервер, на котором уже установлен MySQL
- Для локального запуска Docker не используется — запуск напрямую через `go run` или собранный бинарник
- Конфигурация (токен, DSN для БД и т.п.) — через `.env` файл / переменные окружения

## Функционал

- Регистрация пользователя
- Создание вредной привычки
- Регистрация срыва (с подтверждением)
- Статистика по вредной привычке
- Автообновление основного экрана (~раз в минуту)

### Регистрация пользователя

- При первом запуске бот предлагает зарегистрироваться
- Регистрация сохраняется в базе по id телеграмма, все дальнейшие данные хранятся по нему
- После регистрации бот предлагает создать первую вредную привычку

### Создание вредной привычки

- Пользователь может создать несколько вредных привычек
- Ввод выполняется пошагово — бот задаёт вопросы по одному
- У каждой привычки заполняется:
  - Название
  - Дата и время последнего срыва (точка отсчёта) — в формате `ДД.ММ.ГГГГ ЧЧ:ММ`
  - Стоимость за один срыв (в рублях)
  - Среднее кол-во срывов — вводится численно, пользователь также выбирает период (день / месяц / 3 месяца / полгода / год) через reply-кнопки

### Регистрация срыва

- Пользователь нажимает кнопку с названием привычки на основном экране
- Бот запрашивает подтверждение: «Зарегистрировать срыв по привычке X?» (кнопки «Да» / «Нет»)
- При подтверждении сохраняется время срыва в БД и обновляется время последнего срыва у привычки

### Статистика по вредной привычке

- Пользователь может посмотреть детальную статистику по вредной привычке
- Статистика включает:
  - Название
  - Время последнего срыва
  - Баланс
  - Среднее кол-во срывов за период от точки отсчёта (на реальных данных)
  - Среднее время между срывами

**Формула баланса:**
Потенциальные потери = (среднее кол-во срывов в день × кол-во дней с точки отсчёта × стоимость срыва).
Реальные потери = кол-во зарегистрированных срывов × стоимость срыва.
**Баланс = Потенциальные потери − Реальные потери** (положительный — экономия, отрицательный — перерасход).

**Тренд за последний день:**
Разница между метрикой (баланс / среднее время), посчитанной от начала регистрации до вчера, и той же метрикой, посчитанной от начала регистрации до текущего момента.

## Экраны

### Регистрация

- Приветственное сообщение
- Reply-кнопка «Нажмите чтобы начать»
- После регистрации: «С возвращением» (если пользователь уже был в системе) или «Добро пожаловать» (если новый)
- Если у пользователя есть вредные привычки — показывается «Основной экран»
- Если нет — показывается кнопка «Создать первую вредную привычку»

### Создание вредной привычки

- Пошаговый диалог (бот задаёт вопросы по одному):
  1. Название привычки
  2. Дата и время последнего срыва (`ДД.ММ.ГГГГ ЧЧ:ММ`)
  3. Стоимость за один срыв (рублей)
  4. Среднее кол-во срывов (число)
  5. Период для среднего (reply-кнопки: День / Месяц / 3 месяца / Полгода / Год)
- После завершения: сообщение «Вредная привычка создана»
- Reply-кнопка «Перейти на главную»

### Основной экран

- Текстовое сообщение со статистикой по каждой вредной привычке:
  - Название
  - Время последнего срыва
  - Баланс
  - Среднее кол-во срывов за период от точки отсчёта (реальные данные)
  - Среднее время между срывами
- **Автообновление**: сообщение редактируется раз в ~1 минуту (через edit message)
- Reply-кнопки с названиями привычек (нажатие → регистрация срыва с подтверждением)
- Reply-кнопка «Статистика»
- Reply-кнопка «Добавить привычку»

### Подтверждение срыва

- Сообщение: «Зарегистрировать срыв по привычке [X]?»
- Reply-кнопки: «✅ Да» / «❌ Нет»
- При «Да»: срыв регистрируется, возврат на основной экран
- При «Нет»: возврат на основной экран без изменений

### Статистика по вредной привычке

- Детальная информация:
  - Название
  - Время последнего срыва
  - Баланс + тренд за последний день (↑ / ↓ + дельта)
  - Среднее кол-во срывов за период от точки отсчёта (реальные данные)
  - Среднее время между срывами + тренд за последний день (↑ / ↓ + дельта)
  - История срывов — последние 20
- Reply-кнопка «Назад» (переход на «Основной экран»)

### Ограничения текущей версии

- Удаление и редактирование привычек не реализуется в данной версии
