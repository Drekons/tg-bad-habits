# tg-bad-habits

Telegram-бот для учёта вредных привычек: создание привычек, регистрация срывов, статистика и баланс (потенциальные vs фактические потери).

## Стек

- **Go** 1.24+
- **MySQL**
- **Telegram Bot API** (go-telegram-bot-api)

## Требования

- Go 1.24+
- MySQL (локально или на сервере)
- Токен бота от [@BotFather](https://t.me/BotFather)

## Установка и запуск

1. Клонировать репозиторий:
   ```bash
   git clone git@github.com:Drekons/tg-bad-habits.git
   cd tg-bad-habits
   ```

2. Создать файл `.env` на основе примера:
   ```bash
   cp .env.example .env
   ```
   Заполнить переменные:
   - `BOT_TOKEN` — токен Telegram-бота
   - `DB_DSN` — DSN для MySQL (например: `user:password@tcp(127.0.0.1:3306)/bad_habits?parseTime=true&loc=Local`)
   - `DB_MIGRATIONS_PATH` — путь к миграциям (по умолчанию `internal/migrations`)

3. Применить миграции (при первом запуске БД создаётся через migrate при старте приложения, но схему можно применить и вручную при необходимости).

4. Запуск:
   ```bash
   go run ./cmd/bot
   ```
   Или сборка и запуск бинарника:
   ```bash
   go build -o bot ./cmd/bot
   ./bot
   ```

## Функционал

- **Регистрация** пользователя по Telegram ID
- **Создание привычек** — пошаговый ввод: название, дата/время последнего срыва, стоимость срыва (руб), среднее количество срывов и период (день/месяц/3 мес/полгода/год)
- **Регистрация срыва** — кнопка по привычке → подтверждение → сохранение времени срыва
- **Статистика** — баланс (потенциальные минус фактические потери), среднее число срывов и среднее время между срывами, тренд за последний день, последние 20 срывов
- **Основной экран** — сводка по всем привычкам с автообновлением сообщения примерно раз в минуту

Подробное описание экранов и формул — в [docs/task.md](docs/task.md).

## Структура проекта

- `cmd/bot` — точка входа приложения
- `internal/bot` — логика бота (обработчики, экраны, клавиатуры, состояния)
- `internal/config` — загрузка конфигурации из `.env`
- `internal/db` — подключение к MySQL и миграции
- `internal/models` — модели данных
- `internal/repository` — работа с БД
- `internal/service` — бизнес-логика (привычки, статистика)
- `internal/migrations` — SQL-миграции
- `docs` — документация
